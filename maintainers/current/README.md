# Current Maintainer Docs

This directory holds active maintainer-only material that is still relevant to
ongoing platform work but should not appear in the public `docs/` tree.

Use this area for:
- cutover checklists
- migration plans
- packaging and release-prep checklists
- internal audits
- validation matrices used to gate future cutovers

Public reference docs belong in `docs/`.
Historical or superseded material belongs in `maintainers/archive/`.
